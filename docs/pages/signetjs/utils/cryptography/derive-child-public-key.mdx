# deriveChildPublicKey

The `deriveChildPublicKey` method derives a child public key from a parent public key using the sig.network v1.0.0 epsilon derivation scheme. This is a core function for the Chain Signatures system, allowing one account to securely control multiple accounts across different chains.

## Usage

```ts twoslash
import { utils } from 'signet.js'

// Derive a child public key
const childKey = utils.cryptography.deriveChildPublicKey(
  '04abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890',
  '0x8626f6940E2eb28930eFb4CeF49B2d1F2C9C1199',
  'my_derivation_path',
  utils.constants.KDF_CHAIN_IDS.ETHEREUM
)
```

## Parameters

| Parameter                    | Type                     | Description                                                                            |
| ---------------------------- | ------------------------ | -------------------------------------------------------------------------------------- | --- | --- | --- | --- |
| `rootUncompressedPubKeySEC1` | `UncompressedPubKeySEC1` | The parent public key in uncompressed SEC1 format (e.g., `04                           |     | x   |     | y`) |
| `predecessorId`              | `string`                 | The address of the account calling the signer contract (e.g., EOA or Contract Address) |
| `path`                       | `string`                 | Optional derivation path suffix (defaults to empty string)                             |
| `chainId`                    | `string`                 | The chain ID to use for derivation (e.g., 'ethereum', 'near')                          |

## Returns

| Type                     | Description                                                         |
| ------------------------ | ------------------------------------------------------------------- | --- | --- | --- | --- |
| `UncompressedPubKeySEC1` | The derived child public key in uncompressed SEC1 format (e.g., `04 |     | x   |     | y`) |

## Implementation Details

The method works by:

1. Creating a derivation path string using the format: `sig.network v1.0.0 epsilon derivation,{chainId},{predecessorId},{path}`
2. Computing a scalar value (epsilon) by hashing the derivation path using keccak256 for Ethereum or sha3_256 for NEAR
3. Performing an elliptic curve point addition: `DerivedPubKey = BasePubKey + (epsilon Ã— G)`
4. Returning the derived public key in uncompressed SEC1 format
